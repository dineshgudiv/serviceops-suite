# Contributing

## Branching
- Create feature branches from `main`
- Use conventional commits

## Local Setup
1. Run `./scripts/bootstrap.ps1`
2. Run `./scripts/up.ps1`
3. Verify with `./scripts/proof_serviceops.ps1`

## Pull Requests
- Keep PRs scoped
- Include tests for behavioral changes
- Update docs if API/event contracts change
- Breaking changes require explicit label and migration notes
